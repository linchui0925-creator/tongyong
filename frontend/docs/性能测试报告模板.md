# 性能测试报告

## 一、测试概述

### 1.1 测试目标

评估前端聊天界面在各种场景下的性能表现，包括：

- 消息渲染性能
- 流式输出响应速度
- 大规模消息列表性能
- 内存使用情况
- 网络请求效率

### 1.2 测试环境

```
操作系统：macOS / Windows / Linux
浏览器：Chrome, Firefox, Safari, Edge
设备：桌面电脑、移动设备
网络：WiFi, 4G, 限制网络
```

### 1.3 测试工具

- Chrome DevTools Performance
- Lighthouse
- React Developer Tools
- Web Vitals
- 网络节流工具

## 二、性能指标

### 2.1 核心指标

| 指标 | 目标值 | 优秀值 | 测量方法 |
|------|--------|--------|----------|
| First Contentful Paint (FCP) | < 1.8s | < 1.0s | Lighthouse |
| Largest Contentful Paint (LCP) | < 2.5s | < 1.5s | Lighthouse |
| Time to Interactive (TTI) | < 3.8s | < 2.0s | Lighthouse |
| Cumulative Layout Shift (CLS) | < 0.1 | < 0.05 | Lighthouse |
| First Input Delay (FID) | < 100ms | < 50ms | Lighthouse |

### 2.2 自定义指标

| 指标 | 目标值 | 测试场景 |
|------|--------|----------|
| 消息渲染时间 | < 16ms | 发送消息 |
| 流式输出延迟 | < 100ms | 接收流式内容 |
| 输入响应时间 | < 50ms | 用户输入 |
| 滚动帧率 | ≥ 55fps | 消息列表滚动 |
| 内存占用 | < 200MB | 运行10分钟 |

## 三、测试场景

### 3.1 场景1：首次加载

**测试步骤**:

1. 清除浏览器缓存
2. 打开聊天页面
3. 记录性能指标
4. 重复3次取平均值

**预期结果**:

- FCP < 1.8s
- JS Bundle加载 < 200KB
- 无阻塞资源

**测试代码**:

```javascript
// 使用 Performance API
const performance = window.performance;

performance.mark('page-load-start');

// 页面加载完成后
performance.mark('page-load-end');
performance.measure('page-load', 'page-load-start', 'page-load-end');

const measures = performance.getEntriesByName('page-load');
console.log('Page Load Time:', measures[0].duration);
```

### 3.2 场景2：发送消息

**测试步骤**:

1. 页面加载完成
2. 聚焦输入框
3. 输入测试消息（100字符）
4. 按Enter发送
5. 记录渲染时间

**预期结果**:

- 乐观更新 < 50ms
- 消息DOM创建 < 16ms
- 无布局抖动

**测试代码**:

```javascript
const measureMessageRender = () => {
    const start = performance.now();
    
    // 模拟消息渲染
    setMessages(prev => [...prev, newMessage]);
    
    requestAnimationFrame(() => {
        const end = performance.now();
        console.log('Message Render Time:', end - start);
    });
};
```

### 3.3 场景3：流式输出

**测试步骤**:

1. 发送长文本请求
2. 监控流式输出性能
3. 记录每个chunk的处理时间
4. 测量内存增长

**预期结果**:

- Chunk处理时间 < 50ms
- 帧率保持 > 55fps
- 内存增长 < 10MB/分钟

**测试代码**:

```javascript
const measureStreamingPerformance = () => {
    let lastTime = performance.now();
    let frameCount = 0;
    let totalDelay = 0;
    
    streamChat(message, sessionId, true, {
        onContent: (chunk, fullContent) => {
            const now = performance.now();
            const delay = now - lastTime;
            totalDelay += delay;
            lastTime = now;
            
            // 帧率监控
            frameCount++;
            if (frameCount % 60 === 0) {
                console.log('Average Delay:', totalDelay / frameCount);
                console.log('FPS:', 1000 / (totalDelay / frameCount));
            }
        }
    });
};
```

### 3.4 场景4：大量消息

**测试步骤**:

1. 准备1000条测试消息
2. 渲染消息列表
3. 测量渲染时间和内存占用
4. 测试滚动性能

**预期结果**:

- 初始渲染 < 500ms
- 滚动帧率 > 50fps
- 内存占用 < 150MB

**测试代码**:

```javascript
const measureLargeListPerformance = () => {
    const messages = generateTestMessages(1000);
    const start = performance.now();
    const memoryBefore = performance.memory.usedJSHeapSize;
    
    setMessages(messages);
    
    requestAnimationFrame(() => {
        const end = performance.now();
        const memoryAfter = performance.memory.usedJSHeapSize;
        
        console.log('Render Time:', end - start);
        console.log('Memory Increase:', (memoryAfter - memoryBefore) / 1024 / 1024, 'MB');
    });
};
```

## 四、性能优化建议

### 4.1 代码级优化

#### 4.1.1 React优化

```typescript
// 使用 React.memo 优化消息组件
const MessageItem = React.memo(({ message }) => {
    return <div>{message.content}</div>;
}, (prevProps, nextProps) => {
    return prevProps.message.id === nextProps.message.id &&
           prevProps.message.content === nextProps.message.content;
});

// 使用 useCallback 缓存回调
const handleSendMessage = useCallback(() => {
    // 发送逻辑
}, [inputValue, currentSessionId]);

// 使用 useMemo 缓存计算
const sortedMessages = useMemo(() => {
    return messages.slice().sort((a, b) => a.timestamp - b.timestamp);
}, [messages]);
```

#### 4.3.2 流式优化

```javascript
// 使用 ReadableStream
const stream = await response.body;

// 批量更新UI
let buffer = '';
const BATCH_SIZE = 10;
let batchCount = 0;

for await (const chunk of stream) {
    buffer += chunk;
    batchCount++;
    
    if (batchCount >= BATCH_SIZE) {
        updateUI(buffer);
        batchCount = 0;
        await new Promise(resolve => setTimeout(resolve, 0));
    }
}
```

## 五、内存泄漏检测

### 5.1 常见泄漏场景

1. **未清理的订阅**
   ```typescript
   useEffect(() => {
       const subscription = subscribeToEvents(handleEvent);
       return () => subscription.unsubscribe(); // 清理
   }, []);
   ```

2. **未取消的请求**
   ```typescript
   const abortControllerRef = useRef<AbortController>();
   
   useEffect(() => {
       abortControllerRef.current = new AbortController();
       return () => abortControllerRef.current.abort(); // 取消
   }, []);
   ```

3. **未清理的定时器**
   ```typescript
   useEffect(() => {
       const timer = setInterval(doSomething, 1000);
       return () => clearInterval(timer); // 清理
   }, []);
   ```

1. 打开DevTools → Memory
2. 选择Allocation instrumentation on timeline
3. 执行操作
4. 检查未释放的对象
