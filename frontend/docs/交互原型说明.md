# 交互原型说明

## 一、核心交互流程

### 1.1 发送消息流程

```
用户输入 → 按Enter → 乐观更新（立即显示用户消息）
       ↓
   创建流式连接
       ↓
   实时流式输出 ← → 打字机动画效果
       ↓
   输出完成 → 更新消息状态
```

**详细步骤**:

1. **输入阶段**
   - 用户在输入框中输入文本
   - 输入框自动调整高度（最大150px）
   - 字符计数实时更新（最多10000字符）
   - 发送按钮根据输入状态变化样式

2. **发送阶段**
   - 用户按Enter键（Shift+Enter换行）
   - 立即显示用户消息（乐观更新）
   - 清空输入框
   - 输入框高度重置

3. **响应阶段**
   - 显示助手消息占位符（带流式动画）
   - SSE流式接收响应内容
   - 实时更新消息内容
   - 显示打字光标效果

4. **完成阶段**
   - 隐藏打字光标
   - 更新消息状态为完成
   - 自动滚动到最新消息

### 1.2 流式输出控制

**可中断机制**:

```
流式输出中 → 按Esc键 → 停止生成
                        ↓
                   保存当前内容
                        ↓
                   更新状态为完成
```

**速度控制**:

- 用户可调节打字速度（5-100ms/字符）
- 默认速度：20ms/字符
- 实时预览效果

### 1.3 对话管理流程

#### 1.3.1 创建会话

```
点击"+"按钮 → 显示新建表单 → 输入名称 → 按Enter/点击创建
                                                    ↓
                                              创建成功
                                                    ↓
                                              切换到新会话
                                                    ↓
                                              清空消息列表
```

#### 1.3.2 切换会话

```
点击会话项 → 确认当前会话无流式输出 → 切换会话 → 加载会话消息
```

#### 1.3.3 删除会话

```
点击删除按钮 → 显示确认对话框 → 点击确认 → 删除会话
                                                ↓
                                          刷新会话列表
                                                ↓
                                     如果删除的是当前会话 → 清空消息
```

#### 1.3.4 清空对话

```
点击"清空当前对话" → 显示确认对话框 → 点击确认 → 清空消息列表
```

#### 1.3.5 删除单条消息

```
鼠标悬停消息 → 显示删除按钮 → 点击删除 → 移除消息
```

## 三、交互动画规范

### 3.1 消息动画

#### 3.1.1 消息出现动画

```css
@keyframes slideIn {
    from {
        opacity: 0;
        transform: translateY(20px);
    }
    to {
        opacity: 1;
        transform: translateY(0);
    }
}

/* 持续时间：0.3s */
/* 缓动函数：ease-out */
```

#### 3.1.2 打字光标动画

```css
@keyframes blink {
    0%, 50% {
        opacity: 1;
    }
    51%, 100% {
        opacity: 0;
    }
}

/* 闪烁间隔：1s */
/* 颜色：primary-color */
```

### 3.2 按钮交互

#### 3.2.1 悬停效果

- 缩放：scale(1.05)
- 阴影增强：shadow-lg
- 过渡：250ms ease

#### 3.2.2 按下效果

- 缩放：scale(0.98)
- 过渡：150ms ease

#### 3.2.3 禁用状态

- 透明度：0.5
- 禁用光标：not-allowed

### 3.3 输入框交互

#### 3.3.1 聚焦效果

- 边框颜色变为主色
- 添加光晕：0 0 0 3px rgba(102, 126, 234, 0.1)
- 背景变为白色

#### 3.3.2 自动高度调整

- 最小高度：52px
- 最大高度：150px
- 高度自动根据内容调整

### 3.4 头像菜单

#### 3.4.1 打开动画

```css

/* 持续时间：0.2s */
/* 缓动函数：ease-out */
```

#### 3.4.2 头像选项悬停

- 缩放：scale(1.15)
- 边框变为主色
- 添加光晕

### 3.5 确认对话框

#### 3.5.1 遮罩层

- 背景：rgba(0, 0, 0, 0.5)

#### 3.5.2 对话框出现

## 四、状态指示器

### 4.1 消息状态

| 状态 | 视觉表现 |
|------|----------|
| `sending` | 显示加载动画 |
| `streaming` | 显示打字光标 + 流式内容 |
| `completed` | 正常显示内容 |
| `error` | 显示错误图标 + 红色边框 |

### 4.2 流式状态

| 状态 | 视觉表现 |
|------|----------|
| 空闲 | 发送按钮为渐变色 |
| 流式中 | 停止按钮显示脉冲动画 |
| 加载中 | 加载指示器旋转 |

### 4.3 会话状态

| 状态 | 视觉表现 |
|------|----------|
| 默认 | 白色背景 |
| 悬停 | 灰色背景 + 向右平移 |
| 激活 | 渐变背景色 + 白色文字 |

## 五、错误处理交互

### 5.1 错误提示

**错误Banner**:

```
位置：消息列表上方
动画：从上向下滑入
关闭：点击"×"按钮
持续：直到用户关闭
```

### 5.2 网络错误

**自动重试**:

- 检测到网络错误
- 显示错误消息
- 保留输入内容
- 提示用户检查网络

### 5.3 流式错误

**中断处理**:

- 检测到流式错误
- 显示错误消息
- 保存已接收内容
- 标记消息为错误状态

## 九、动画性能优化

### 9.1 GPU加速

使用GPU加速的属性：

- `transform`
- `opacity`
- `filter`（在必要时）

避免动画的属性：

- `width`、`height`
- `margin`、`padding`
- `top`、`left`

### 9.2 减少重绘

```css
.message-item {
    will-change: transform, opacity;
    transform: translateZ(0);  /* 强制GPU加速 */
}
```

### 9.3 动画帧率

目标帧率：60fps

- 动画时长：≤300ms
- 使用`requestAnimationFrame`
- 避免布局抖动
