# TongYong Agent 记忆系统重构文档

## 一、重构概述

### 1.1 解决的问题

1. **记忆注入机制缺陷**：记忆没有在对话开始前正确加载
2. **会话设定缺失**：用户偏好和设置没有被作为永久设定保存
3. **记忆管理不完整**：缺少更新、删除、查询功能
4. **记忆验证缺失**：无法验证记忆是否正确加载
5. **版本控制缺失**：无法追踪记忆的变更历史

### 1.2 重构目标

1. ✅ 实现会话记忆持久化机制
2. ✅ 增强记忆流管理模块（增删改查）
3. ✅ 实现记忆注入机制
4. ✅ 添加记忆验证功能
5. ✅ 建立记忆版本控制

## 二、架构变更

### 2.1 数据模型变更

#### Memory 模型增强

**新增字段**：
- `updated_at`: 更新时间戳
- `version`: 版本号（用于版本控制）

#### 新增 memory_settings 表

```sql
CREATE TABLE memory_settings (
    id TEXT PRIMARY KEY,
    session_id TEXT NOT NULL,
    key TEXT NOT NULL,
    value TEXT NOT NULL,
    type TEXT NOT NULL,
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL,
    FOREIGN KEY (session_id) REFERENCES sessions(id),
    UNIQUE(session_id, key)
);
```

**用途**：存储会话级别的永久设定，如用户偏好、回复风格等

### 2.2 核心组件变更

#### AgentEngine 增强

**新增方法**：

1. `_load_session_settings(session_id)` - 加载会话设定
   - 加载会话设定（从 memory_settings 表）
   - 加载操作习惯记忆（作为永久设定）
   - 加载分析结论记忆（作为永久设定）
   - 加载关键决策记忆（作为永久设定）

2. `_verify_memory_injection(memories)` - 验证记忆注入
   - 检查每条记忆是否正确注入到 context
   - 统计记忆类型分布
   - 返回验证结果

3. `verify_memory_loading(session_id)` - 验证记忆加载情况
   - 返回所有持久化记忆的详细信息
   - 统计总数和分类

4. `update_memory()` - 更新记忆
   - 支持更新内容和重要性
   - 自动增加版本号
   - 重新向量化存储

5. `delete_memory()` - 删除记忆
   - 从数据库删除
   - 从向量库删除

6. `add_setting()` / `get_setting()` / `update_setting()` / `delete_setting()` - 会话设定管理

7. `get_memory_versions()` - 获取记忆版本历史

## 三、核心功能实现

### 3.1 记忆注入机制

**重构前的问题**：
- 记忆在用户消息之后才检索
- 会话设定没有被作为永久设定加载
- 每次对话都重新加载，导致记忆丢失

**重构后的流程**：

```python
async def chat(self, session_id, message, use_memory=True):
    # 1. 加载会话设定（永久设定）
    await self._load_session_settings(session_id)
    
    # 2. 检索相关记忆
    if use_memory and self.llm:
        embedding = await self.llm.get_embedding(message)
        memories = await self.vector_store.search(
            message, embedding, k=5, session_id=session_id
        )
        if memories:
            for memory in memories:
                self.context.add_message("system", f"[相关记忆] {memory.content}")
    
    # 3. 添加用户消息
    self.context.add_message("user", message)
    
    # 4. 对话生成
    response = await self.llm.chat(messages=self.context.get_messages())
    
    # 5. 验证记忆注入
    verification = await self._verify_memory_injection(memories)
```

**关键改进**：
- 在对话开始前就加载所有会话设定
- 操作习惯、分析结论、关键决策作为永久设定
- 每次对话都会重新加载这些设定
- 添加记忆验证功能确保注入正确

### 3.2 记忆类型处理

| 类型 | 加载时机 | 加载数量 | 说明 |
|------|----------|----------|------|
| 会话设定 | 每轮对话开始 | 全部 | 用户偏好、回复风格等 |
| 操作习惯 | 每轮对话开始 | 最多3条 | 用户偏好、操作模式 |
| 分析结论 | 每轮对话开始 | 最多2条 | 数据洞察、报告结论 |
| 关键决策 | 每轮对话开始 | 最多2条 | 重要决定、业务选择 |
| 相关记忆 | 用户消息后 | 最多5条 | 基于向量检索的相关记忆 |

## 五、数据库迁移

### 5.1 添加新列到 memories 表

如果现有数据库没有 updated_at 和 version 列，执行以下 SQL：

```sql
ALTER TABLE memories ADD COLUMN updated_at TEXT;
ALTER TABLE memories ADD COLUMN version INTEGER DEFAULT 1;
```

### 5.2 创建 memory_settings 表

```sql
CREATE TABLE IF NOT EXISTS memory_settings (
    id TEXT PRIMARY KEY,
    session_id TEXT NOT NULL,
    key TEXT NOT NULL,
    value TEXT NOT NULL,
    type TEXT NOT NULL,
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL,
    FOREIGN KEY (session_id) REFERENCES sessions(id),
    UNIQUE(session_id, key)
);

CREATE INDEX IF NOT EXISTS idx_settings_session_id ON memory_settings(session_id);
```

## 六、性能优化

### 6.1 索引策略

```sql
-- 会话设定索引
CREATE INDEX idx_settings_session_id ON memory_settings(session_id);

-- 记忆表已有索引
CREATE INDEX idx_memories_session_id ON memories(session_id);
CREATE INDEX idx_memories_type ON memories(type);
CREATE INDEX idx_memories_importance ON memories(importance DESC);
```

### 6.2 加载限制

为了避免 context 过长，对永久设定的加载进行了限制：

- 会话设定：全部加载
- 操作习惯：最多3条
- 分析结论：最多2条
- 关键决策：最多2条
- 相关记忆：最多5条

## 七、已知限制

### 7.1 Token 限制

- 每次对话都会加载所有永久设定，可能导致 token 消耗增加
- 建议在高并发场景下监控 token 使用情况

### 7.2 向量库性能

- 向量库在数据量超过10000条时可能需要优化
- 建议定期清理过期的对话记忆

### 7.3 会话隔离

- 当前实现中，共享记忆可以被所有会话检索
- 如果需要严格隔离，需要修改 search 方法

---

## 八、后续优化建议

### 8.1 短期优化

1. **记忆压缩**：当记忆数量过多时，使用 LLM 压缩历史记忆
2. **增量加载**：只加载新增的设定，而不是每次都全部加载
3. **缓存机制**：在内存中缓存会话设定，减少数据库查询

### 8.2 长期优化

1. **分布式记忆**：支持跨会话共享记忆
2. **记忆分层**：区分短期记忆、长期记忆、永久记忆
3. **自动摘要**：定期自动生成会话摘要
4. **记忆推荐**：基于用户行为推荐可能需要的记忆
