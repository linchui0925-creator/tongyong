# TongYong Agent 记忆系统修改优化文档

## 一、当前问题总结

### 1.1 会话切换问题
| 问题 | 原因 | 影响 |
|------|------|------|
| 切换会话后返回对话没有变化 | 前端不传 session_id，后端不加载/保存会话消息 | 用户感受不到会话隔离 |
| 每轮对话都是新 ContextManager | agent.py 每次请求新建 context | 历史消息不持久化 |

### 1.2 记忆忘记问题
| 问题 | 原因 | 影响 |
|------|------|------|
| messages 表是空的 (0条) | 每次对话没有存库 | 对话历史完全丢失 |
| ChromaDB 是空的 | 没有自动向量化存储 | 无法语义检索历史 |
| 记忆检索返回全局结果 | VectorStore.search() 没有按 session_id 过滤 | 会话A的记忆出现在会话B |

### 1.3 架构缺陷
```
当前流程:
用户发消息 → AgentEngine.chat() → 空 ContextManager → LLM → 返回响应
                                                      ↑
                                            ❌ 没有加载历史

正确流程应该是:
用户发消息 → AgentEngine.chat(session_id)
           → MemoryStorage.get_messages(session_id) 加载历史
           → 添加新消息到 ContextManager
           → LLM 生成响应
           → 消息存 messages 表
           → 向量化存 ChromaDB
```

```python
# 修改后代码
async def chat(
    self,
    session_id: Optional[str],
    message: str,
    use_memory: bool = True
) -> Dict[str, Any]:
    # 1. 加载会话历史消息 (新增)
    if session_id:
        historical_messages = await self.memory_storage.get_messages(session_id)
        for msg in historical_messages:
            self.context.add_message(msg.role, msg.content)

    # 2. 添加当前用户消息
    self.context.add_message("user", message)

    # 3. 检索相关记忆 (可选，session_id 过滤)
    memories = []
    if use_memory and self.llm:
        try:
            embedding = await self.llm.get_embedding(message)
            # 按 session_id 过滤检索 (修改 vector.py)
            memories = await self.vector_store.search(
                message, embedding, k=3, session_id=session_id
            )
            if memories:
                context = "\n".join([m.content for m in memories])
                self.context.add_message("system", f"相关记忆：\n{context}")
        except Exception:
            pass

    # 4. 调用 LLM
    response = "智能体已收到消息"
    if self.llm:
        try:
            response = await self.llm.chat(
                messages=self.context.get_messages()
            )
        except Exception:
            response = "智能体已收到消息"

    # 5. 添加助手回复
    self.context.add_message("assistant", response)

    # 6. 保存消息到数据库 (新增)
    if session_id:
        await self.memory_storage.add_message(session_id, "user", message)
        await self.memory_storage.add_message(session_id, "assistant", response)

        # 7. 向量化存储 (新增)
        if self.llm:
            try:
                user_embedding = await self.llm.get_embedding(message)
                await self.vector_store.add_text(
                    session_id, "user", message, user_embedding
                )
                assistant_embedding = await self.llm.get_embedding(response)
                await self.vector_store.add_text(
                    session_id, "assistant", response, assistant_embedding
                )
            except Exception:
                pass

    return {
        "reply": response,
        "session_id": session_id,
        "memory_added": [m.dict() for m in memories],
        "tools_used": []
    }
```

## 六、记忆迭代机制（第六阶段）

### 6.1 设计目标

复刻 Claude-Mem 的记忆迭代机制：
- 批量审查：积累多轮对话后，用本地大模型审查是否重复
- 提炼精要：本地模型直接输出精要内容
- 自动判断类型：自动识别是操作习惯/分析结论/关键决策等
- Skill 触发：作为系统 skill 注入到主模型提示词

### 6.2 触发机制

| 条件 | 说明 |
|------|------|
| 固定轮数 | 每积累 N 轮对话后触发审查（建议 N=5） |
| token 阈值 | 对话总 token 超过阈值时触发 |

### 6.3 审查流程

```
用户发消息 → 对话 → 积累 N 轮 → 本地大模型审查
                                          ↓
                                    判断是否重复：
                                    ├─ 有重复 → 丢弃（可选择更新重要性）
                                    └─ 无重复 → 提炼精要 → 存储类型自动判断 → 存入向量库
```

### 6.4 本地大模型审查 Skill

```python
# 记忆审查 Skill Prompt
MEMORY_REVIEW_SKILL = """
## 记忆审查 Skill

你是一个记忆审查助手。当用户对话达到一定轮数时，你需要审查对话内容并决定是否需要存储为长期记忆。

### 你的任务
1. 判断当前对话是否包含值得存储的新知识
2. 如果是，提炼出精要内容
3. 自动判断记忆类型

### 记忆类型分类
- 操作习惯：用户偏好、操作模式、常用功能
- 分析结论：数据洞察、报告结论、发现
- 关键决策：重要决定、业务选择
- 其他：不属于上述的内容

### 输出格式
请直接输出以下 JSON 格式，不要其他内容：
{
    "need_store": true/false,  # 是否需要存储
    "type": "操作习惯/分析结论/关键决策/其他",  # 记忆类型
    "content": "精要内容（如果不需存储则为空）",  # 提炼的精要
    "reason": "判断原因"  # 为什么存储或不存储
}

### 规则
- 只提取之前记忆中没有的新信息
- 已有的类似内容不重复存储
- 重要性高的内容优先存储
"""

# 用法示例
async def review_and_extract(
    conversation_history: List[dict],
    existing_memories: List[str],
    local_llm

    审查并提取记忆

    Args:

        existing_memories: 已有的记忆列表（用于去重对比）

{MEMORY_REVIEW_SKILL}

{format_memories(existing_memories)}

请审查并输出 JSON 结果。
"""
    result = await local_llm.chat(prompt)
    return parse_json(result)
```

```python
class AgentEngine:
    def __init__(self, llm=None, local_llm=None):
        self.llm = llm
        self.local_llm = local_llm  # 本地大模型（用于记忆审查）
        self.conversation_buffer = []  # 对话缓冲
        self.REVIEW_THRESHOLD = 5  # 多少轮审查一次

    async def chat(self, session_id, message, use_memory=True):
        # ... 现有对话逻辑 ...

        # 添加到对话缓冲
        self.conversation_buffer.append({
            "role": "user",
            "content": message
        })
        self.conversation_buffer.append({
            "role": "assistant",
            "content": response
        })

        # 检查是否达到审查阈值
        if len(self.conversation_buffer) >= self.REVIEW_THRESHOLD * 2:
            await self._review_and_store(session_id)
            self.conversation_buffer = []  # 清空缓冲

    async def _review_and_store(self, session_id: str):
        """审查并存储记忆"""
        if not self.local_llm:
            return

        # 获取已有记忆
        existing = await self.vector_store.get_all_by_session(session_id)

        # 调用本地大模型审查
        result = await review_and_extract(
            self.conversation_buffer,
            [m.content for m in existing],
            self.local_llm
        )

        if result.get("need_store"):
            # 存储新记忆
            memory = Memory(
                id=str(uuid4()),
                type=result["type"],  # 自动判断的类型
                content=result["content"],
                importance=2,
                session_id=session_id,
                created_at=datetime.now().isoformat()
            )
            await self.memory_storage.add_memory(memory)

            # 向量化存储
            embedding = await self.llm.get_embedding(result["content"])
            await self.vector_store.add(memory, embedding)
```

## 七、风险和注意事项

### 7.1 Token 限制
- 每次加载历史会消耗 token，建议在 ContextManager 中限制最大历史条数
- 当前 ContextManager 有 `max_tokens=4000` 限制，需确认生效

### 7.2 向量库性能
- ChromaDB 默认是内存模式，首次调用会加载慢
- 已设置 persist_directory 持久化

### 7.3 错误处理
- LLM 调用失败时不影响消息存库
- 向量化失败时不影响对话流程
