# TongYong Agent 开发计划（详细版）

## 一、项目概述

| 维度 | 内容 |
|------|------|
| **项目名称** | TongYong Agent（通用智能体） |
| **目标用户** | 运营人员（小项目，仅内部使用） |
| **核心价值** | 通过不断迭代记忆，让智能体越来越懂业务、越来越像用户 |
| **记忆框架** | Claude-Mem 架构（核心功能，必须做） |
| **数据处理** | 除视频外所有文本、图片类都能识别 → 可视化图表 |

### 第二阶段：核心引擎 + 记忆模块（核心）

| 任务 | 子任务 | 验收标准 |
|------|-------|----------|
| 2.1 核心引擎骨架 | AgentEngine、ContextManager | 代码可运行 |
| 2.2 SQLite 表设计 | sessions、messages、memories 表 | 建表成功 |
| 2.3 ChromaDB 集成 | 向量存储、相似搜索 | 可存储、可搜索 |

| 2.6 记忆查看器 | 前端卡片展示 | 可查看记忆流 |

**技术要点 - 记忆压缩提示词**：
```
你是一个记忆提炼助手。请从以下对话中提炼出新的知识点：
- 用户操作习惯（偏好什么图表、什么时候活跃）
- 关键决策（做了什么决定、为什么）
- 重要结论（分析结果、报告结论）

规则：
1. 只提取之前记忆中没有的新信息
2. 已有的类似内容不重复存储
3. 输出格式：JSON 数组，每项含 type、content、importance

对话内容：
{conversation}
```

**验收方式**：
- 创建会话 → 保存消息 → 搜索记忆 → 返回结果
- 重复内容不重复存入向量库

### 第五阶段：记忆迭代（核心）

| 任务 | 子任务 | 验收标准 |
|------|-------|----------|
| 5.1 操作习惯记忆 | 记录用户偏好 | 记住偏好 |
| 5.2 分析结果记忆 | 记住历史结论 | 可回顾 |
| 5.3 记忆检索优化 | 自然语言搜索 | 搜索准确 |
| 5.4 记忆可视化 | 记忆时间线 | 显示完整 |
| 5.5 **记忆迭代机制** | **本地大模型审查去重** | **自动提炼精要** |

**记忆类型**：
| 类型 | 内容 | 记忆时机 |
|------|------|----------|
| 操作习惯 | 偏好图表、上传时间 | 用户操作时 |
| 分析结论 | 报告结论、数据洞察 | 分析完成后 |
| 关键决策 | 做了什么决定、为什么 | 用户确认时 |

**5.5 记忆迭代机制（核心）**：
| 维度 | 内容 |
|------|------|
| 触发方式 | 批量审查（建议每5轮对话后触发） |
| 审查模型 | 本地大模型（如 Qwen2.5） |
| 审查流程 | 判断是否重复 → 提炼精要 → 自动判断类型 → 存入向量库 |
| 存储类型 | 自动判断（操作习惯/分析结论/关键决策/其他） |
| Skill | 记忆审查 Skill 注入到主模型提示词 |

**记忆审查 Skill 说明**：
- 本地大模型判断对话是否包含新知识
- 如果没有重复，提炼精要内容
- 自动判断记忆类型（操作习惯/分析结论/关键决策/其他）
- 输出 JSON 格式结果

**验收方式**：
- 发送5轮对话 → 本地大模型自动审查 → 检查记忆是否正确存储
- 尝试发送类似内容 → 验证不重复存储

## 四、接口约定

### 4.1 对话接口

```bash
# POST /api/chat
Request:
{
    "session_id": "可选，创建新会话时不传",
    "message": "用户消息",
    "use_memory": true
}

Response:
{
    "reply": "AI回答",
    "session_id": "会话ID",
    "memory_added": ["新记忆项"],
    "tools_used": ["使用的工具"]
}
```

### 4.2 记忆接口

```bash
# GET /api/memory/{session_id}
Response:
{
    "memories": [
        {"type": "操作习惯", "content": "偏好饼图", "created_at": "时间"}
    ]
}

# POST /api/memory/search
Request:
{"query": "自然语言搜索"}

Response:
{"results": [...]}
```

### 4.3 数据接口

```bash
# POST /api/data/upload
Request:
{
    "type": "file|url|clipboard",
    "data": "文件或URL或base64"
}

Response:
{
    "table_id": "表格ID",
    "columns": [...],
    "rows": [...]
}
```

### 4.4 图表接口

```bash
# POST /api/chart/generate
Request:
{
    "table_id": "表格ID",
    "chart_types": ["pie", "bar"]  # 用户选择的类型
}

Response:
{
    "charts": [
        {"type": "pie", "data": ...}
    ]
}
```
