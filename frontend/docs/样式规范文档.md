# 样式规范文档

## 一、设计系统概述

本样式规范基于现代设计美学原则，采用CSS变量系统，支持响应式布局和交互动画。

## 二、色彩系统


## 六、动画系统

### 6.1 过渡时间

```css
:root {
    --transition-fast: 150ms ease;     /* 快速 - 微交互 */
    --transition-base: 250ms ease;    /* 基础 - 标准过渡 */
    --transition-slow: 350ms ease;     /* 慢速 - 大型动画 */
}
```

### 6.2 核心动画

#### 6.2.1 消息滑入动画

```css
@keyframes slideIn {
    from {
        opacity: 0;
        transform: translateY(20px);
    }
    to {
        opacity: 1;
        transform: translateY(0);
    }
}

.message-item {
    animation: slideIn 0.3s ease-out;
}
```

#### 6.2.2 打字光标闪烁

```css
@keyframes blink {
    0%, 50% {
        opacity: 1;
    }
    51%, 100% {
        opacity: 0;
    }
}

.typing-cursor {
    animation: blink 1s infinite;
    color: var(--primary-color);
}
```

#### 6.2.3 浮动动画（空状态图标）

```css
@keyframes float {
    0%, 100% {
        transform: translateY(0);
    }
    50% {
        transform: translateY(-10px);
    }
}

.empty-icon {
    animation: float 3s ease-in-out infinite;
}
```

#### 6.2.4 脉冲动画（停止按钮）

```css

    0%, 100% {
        opacity: 1;
    }
    50% {
        opacity: 0.8;
    }

    animation: pulse 2s infinite;
}
```

#### 6.2.5 旋转动画（加载指示器）

```css

    from {
        transform: rotate(0deg);
    }
    to {
        transform: rotate(360deg);
    }
```


## 九、阴影和层次

### 9.1 阴影使用场景

| 场景 | 阴影级别 | 说明 |
|------|----------|------|
| 按钮默认 | `shadow-sm` | 微妙的阴影 |
| 卡片/消息气泡 | `shadow-md` | 中等阴影 |
| 下拉菜单/弹出框 | `shadow-lg` | 明显的阴影 |
| 模态对话框 | `shadow-xl` | 最强阴影 |

### 9.2 Z-index层级

```css
/* 层级顺序（从低到高） */

    z-index: 1;
}

.avatar-menu {
    z-index: 1000;           /* 头像菜单 */
}

.confirm-overlay {
    z-index: 9999;           /* 确认对话框遮罩 */
}
```

## 十一、无障碍设计

### 11.1 焦点样式

```css
/* 自定义焦点轮廓 */
button:focus-visible,
input:focus-visible,
textarea:focus-visible {
    outline: 2px solid var(--primary-color);
    outline-offset: 2px;
}
```

### 11.2 对比度

确保文本和背景的对比度符合WCAG 2.1 AA标准：
- 主要文本：至少4.5:1对比度
- 大文本（18px+）：至少3:1对比度
- UI组件和图形：至少3:1对比度

## 十二、代码规范

### 12.1 CSS编写规范

1. **使用CSS变量**：所有可复用的值定义为CSS变量
2. **语义化命名**：使用有意义的类名
3. **模块化组织**：按组件组织样式
4. **注释清晰**：标注重要区块
5. **遵循BEM**：块__元素--修饰符

## 十三、最佳实践

1. **优先使用CSS变量**：提高代码复用性和维护性
2. **移动优先设计**：从最小屏幕开始，逐步增强
3. **使用Flexbox/Grid**：现代布局技术
4. **避免内联样式**：保持样式与结构分离
5. **优化动画性能**：使用transform和opacity
6. **测试跨浏览器**：确保各浏览器兼容性
7. **保持一致性**：统一的间距、颜色、字体
