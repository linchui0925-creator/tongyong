# 组件设计文档

## 一、概述

本文档描述了TongYong Agent前端组件的设计和架构，包括组件层次结构、状态管理、Props接口定义以及组件间的通信机制。

## 二、组件架构

### 2.1 组件层次结构

```
App
└── ModernChatPanel (主聊天容器)
    ├── ChatSidebar (侧边栏)
    │   ├── SidebarHeader (头部)
    │   ├── NewSessionForm (新建会话表单)
    │   ├── SessionList (会话列表)
    │   │   └── SessionItem (会话项)
    │   └── SidebarFooter (底部)
    ├── ChatMain (主聊天区域)
    │   ├── ChatHeader (头部)
    │   ├── ErrorBanner (错误提示)
    │   ├── ChatMessages (消息列表)
    │   │   ├── EmptyState (空状态)
    │   │   └── MessageItem (消息项)
    │   │       ├── MessageAvatar (头像)
    │   │       ├── AvatarMenu (头像选择菜单)

    │       └── InputContainer (输入容器)

```

### 2.2 组件说明

#### 2.2.1 ModernChatPanel

**功能**: 主聊天面板容器组件，负责整体布局和状态管理

**Props接口**:
```typescript
interface ModernChatPanelProps {
    initialSessionId?: string;  // 初始会话ID
}
```

**状态管理**:
- `messages`: 消息列表
- `inputValue`: 输入框内容
- `sessions`: 会话列表
- `currentSessionId`: 当前会话ID
- `isStreaming`: 是否正在流式输出
- `isLoading`: 是否正在加载
- `errorMessage`: 错误消息

**核心功能**:
1. 乐观更新：用户发送消息后立即显示，无需等待响应
2. 流式输出：支持SSE实时流式输出，带打字机效果
3. 对话管理：创建、删除、清空会话
4. 消息管理：删除单条消息
5. 键盘快捷键支持

#### 2.2.2 ChatSidebar

**功能**: 侧边栏组件，显示会话列表

**状态**:
- `sessions`: 会话列表
- `currentSessionId`: 当前会话ID
- `showNewSession`: 是否显示新建会话表单

**交互功能**:
1. 显示所有会话
2. 切换会话
3. 创建新会话
4. 删除会话（带确认对话框）
5. 清空当前对话

#### 2.2.3 ChatMessages

**功能**: 消息列表组件

**Props接口**:
```typescript
interface ChatMessagesProps {
    messages: Message[];  // 消息列表
    onDeleteMessage?: (messageId: string) => void;  // 删除消息回调
}
```

**渲染逻辑**:
1. 空状态：显示欢迎信息和快捷键说明
2. 消息列表：按时间顺序渲染所有消息
3. 流式状态：显示打字光标动画

#### 2.2.4 MessageItem

**功能**: 单条消息组件

**Props接口**:
```typescript
interface MessageItemProps {
    message: Message;  // 消息对象
    avatar: AvatarConfig;  // 头像配置
    onAvatarClick?: () => void;  // 头像点击回调
    onDelete?: () => void;  // 删除回调
}
```

**消息状态**:
- `sending`: 发送中
- `streaming`: 流式输出中
- `completed`: 完成
- `error`: 错误

**样式变化**:
- 用户消息：右对齐，渐变背景色
- 助手消息：左对齐，白色背景
- 流式消息：带打字光标动画
- 错误消息：显示错误图标

#### 2.2.5 ChatInputArea

**功能**: 输入区域组件

**状态**:
- `inputValue`: 输入内容
- `isStreaming`: 是否正在流式输出

**功能**:
1. 自动调整输入框高度
2. 支持多行输入
3. 字符计数显示
4. 发送按钮状态管理
5. 快捷键支持（Enter发送，Shift+Enter换行）

#### 2.2.6 ConfirmDialog

**功能**: 确认对话框组件

**Props接口**:
```typescript
interface ConfirmDialogProps {
    title: string;  // 标题
    message: string;  // 消息内容
    confirmText?: string;  // 确认按钮文本
    cancelText?: string;  // 取消按钮文本
    onConfirm: () => void;  // 确认回调
    onCancel?: () => void;  // 取消回调
    type?: 'warning' | 'danger' | 'info';  // 类型
}
```

**交互**:
1. 点击遮罩层关闭
2. ESC键关闭
3. 按钮点击触发回调
4. 动画过渡效果

## 四、组件通信机制

### 4.1 父子组件通信

使用React Props进行父子组件通信：
- 父组件通过Props传递数据和回调函数
- 子组件通过回调函数向父组件传递事件

### 4.2 状态提升

所有共享状态提升到ModernChatPanel组件：
- 消息列表
- 会话列表
- 当前会话ID
- 流式状态
- 错误状态

### 4.3 回调函数模式

```typescript
// 发送消息回调
const handleSendMessage = useCallback(async () => {
    // 乐观更新：立即显示用户消息
    setMessages(prev => [...prev, userMessage]);
    
    // 创建流式连接
    abortControllerRef.current = streamChat(message, sessionId, true, {
        onStart: () => setIsStreaming(true),
        onContent: (chunk, full) => {
            setMessages(prev => prev.map(msg => 
                msg.id === assistantMessageId 
                    ? { ...msg, content: full }
                    : msg
            ));
        },
        onDone: () => setIsStreaming(false),
        onError: (error) => setErrorMessage(error)
    });
}, [inputValue, currentSessionId]);
```

## 五、性能优化策略

### 5.1 React优化

1. **useCallback**: 缓存回调函数，减少不必要的重新渲染
2. **useMemo**: 缓存计算结果
3. **React.memo**: 避免不必要的子组件重渲染
4. **虚拟滚动**: 消息列表使用虚拟滚动优化大量消息

### 5.2 消息渲染优化

```typescript
// 使用唯一key优化列表渲染
messages.map((msg) => (
    <MessageItem 
        key={msg.id}  // 使用唯一ID而非索引
        message={msg}
    />
));
```

### 5.3 懒加载

```typescript
// 代码分割
const ConfirmDialog = React.lazy(() => import('./ConfirmDialog'));
```

## 七、错误处理

### 7.1 错误边界

```typescript
class ErrorBoundary extends React.Component {
    componentDidCatch(error, errorInfo) {
        console.error('组件错误:', error, errorInfo);
        this.setState({ hasError: true });
    }
}
```

### 7.2 网络错误处理

```typescript
// API错误处理
try {
    await streamChat(message);
} catch (error) {
    if (error.name === 'AbortError') {
        console.log('请求已中断');
    } else {
        setErrorMessage('网络错误');
    }
}
```
